Data Design — System Prompt Approach

Payment data is embedded directly in the ChatClient system prompt in AiConfig.java. PaymentDataTools.java exists in the codebase with full @Tool annotations but is not registered with the ChatClient on this deployment — the GTO-models proxy on dhaka returns 400 when the OpenAI tools field is present. The payflow-ai-epc variant targets a foundation with native tool calling support.

Chat Request — SSE Streaming Lifecycle

1
User sends message
React calls POST /api/chat/stream with { message, sessionId }. Server opens an SSE connection. If sessionId is absent (first message) the server generates a UUID and sends it as the final SSE event.
SSE / EventSourcesessionId
2
HTTP 200 committed immediately
emitter.send(SseEmitter.event().comment("connected")) fires before the AI call. This commits the response headers to the client — if the AI fails fast, Spring returns an SSE error event rather than a 503 JSON error from BasicErrorController. A keepalive comment is also sent every 15 s to prevent CF GoRouter's 180 s idle timeout.
SseEmitterkeepalive/15sNo 503
3
Conversation history prepended
MessageChatMemoryAdvisor looks up the sessionId in InMemoryChatMemory and prepends prior messages before the prompt is sent to the LLM. Each session has independent history. The compact system prompt (~150 tokens) with embedded payment data is always prepended as the system message.
InMemoryChatMemoryMessageChatMemoryAdvisor
4
LLM call — Devstral streams tokens
Spring AI sends the prompt (system + history + user message) to Devstral-Small-2507 via the OpenAI-compatible streaming endpoint. No tools field is included — the GTO-models proxy does not support it. The model generates a response grounded in the system prompt data.
Devstral-Small-2507OpenAI SSE protocolNo tools field
5
Tokens streamed to browser in real-time
Spring AI's Flux<String> is iterated; each token is forwarded as an SSE data: event. React appends each token to the message bubble as it arrives. On completion, an event:done data:<sessionId> event is sent. On error, an event:error data:<message> event is sent.
Flux<String>.toIterable()React token append

Key Source Files

⚙️
AiConfig.java
config/AiConfig.java
Creates the ChatClient bean with the compact system prompt (payment data embedded, ~150 tokens) and MessageChatMemoryAdvisor for per-session memory. No tools registered — proxy limitation.
Spring AI config
🔌
GenAiVcapPostProcessor.java
config/GenAiVcapPostProcessor.java
Reads VCAP_SERVICES["ai-models"][0] at startup. Maps credentials.endpoint.openai_api_base and api_key to spring.ai.openai.chat.base-url and chat.api-key. Does nothing locally.
CF VCAP binding
🔧
RedisSslFixPostProcessor.java
config/RedisSslFixPostProcessor.java
Fixes a java-cfenv-boot 3.1.x incompatibility: it sets spring.data.redis.ssl=true (String) but Spring Boot 3.3+ expects a nested Ssl record with spring.data.redis.ssl.enabled. Adds the correct key at highest priority.
Compat fix
🔧
PaymentDataTools.java
tools/PaymentDataTools.java
Six @Tool-annotated methods returning mock payment data. Present in the codebase but not registered with ChatClient on this deployment. Available for activation on payflow-ai-epc (tool-calling capable foundation).

controller/ChatController.java
