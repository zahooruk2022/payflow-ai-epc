Data & Tool Calling — PaymentDataTools

The EPC foundation's chat-and-tools-model plan supports the OpenAI tools field. PaymentDataTools.java exposes six @Tool-annotated methods that gpt-oss:20b can invoke during inference — the model decides which tools to call based on the user's question. Transaction data is stored in PostgreSQL (via payflow-postgres CF service). Fraud sliding-window counters are maintained in Redis (via payflow-redis). Async payment processing flows through RabbitMQ.

@Tool methods — PaymentDataTools.java · called by gpt-oss:20b at inference time

getRecentTransactions — queries PostgreSQL for latest interbank payments with risk scores, status, timestamps
Statistics (24h)
getPaymentStatistics — aggregates 247 txns, £4.82M volume, 18 fraud flags (7.3%), avg risk 34
Account Balances
getAccountBalances — Albion £5M · Meridian £3.5M · Crestfield £2.75M · Harrington £4.2M · Caledonian £1.8M · Vantage £6.1M
Fraud Alerts

(int limit) → String
Returns the most recent payment transactions with sender, receiver, amount, status, and risk score. Model calls this when asked about recent activity.

24-hour KPI summary: total transactions, total volume, flagged count, fraud rate, peak hour, highest single payment.

Current fraud alerts with rule name, amount, risk score, sender account, and trigger timestamp.

Documents the four fraud rules with thresholds: HIGH_AMOUNT, RAPID_SUCCESSION (Redis counters), ROUND_NUMBER, HIGH_RISK_ACCOUNT.

Generates a detailed investigation narrative for a transaction — timeline, risk factors, triggered rules, recommended action.

Chat Request — SSE Streaming Lifecycle

1
User sends message
React calls POST /api/chat/stream with { message, sessionId }. Server opens an SSE connection. If sessionId is absent (first message) the server generates a UUID and sends it as the final SSE event.
SSE / EventSourcesessionId
2
HTTP 200 committed immediately
emitter.send(SseEmitter.event().comment("connected")) fires before the AI call. This commits the response headers to the client — if the AI fails fast, Spring returns an SSE error event rather than a 503 JSON error. A keepalive comment is also sent every 15 s to prevent CF GoRouter's 180 s idle timeout.
SseEmitterkeepalive/15sNo 503
3
Conversation history prepended
MessageChatMemoryAdvisor looks up the sessionId in InMemoryChatMemory and prepends prior messages before the prompt is sent to the LLM. Each session has independent history. A compact system prompt establishes the AI's role as a payment analyst.
InMemoryChatMemoryMessageChatMemoryAdvisor
4
LLM call — gpt-oss:20b processes and may invoke tools
Spring AI sends the prompt (system + history + user message + tool definitions) to gpt-oss:20b via the OpenAI-compatible streaming endpoint. The model decides whether to call one or more @Tool methods (e.g. getRecentTransactions, getFraudAlerts) before generating its final response. Tool calls are executed by Spring AI and results are returned inline to the model.
gpt-oss:20b@Tool invocationPostgreSQL · Redis
5
Tokens streamed to browser in real-time
Spring AI's Flux<String> is iterated; each token is forwarded as an SSE data: event. React appends each token to the message bubble as it arrives. On completion, an event:done data:<sessionId> event is sent. On error, an event:error data:<message> event is sent.
Flux<String>.toIterable()React token append

AiConfig.java

Six @Tool-annotated methods returning payment data: recent transactions, statistics, fraud alerts, account balances, fraud rules explanation, investigation narratives. Registered with ChatClient on EPC via defaultTools().

GenAiVcapPostProcessor.java
config/GenAiVcapPostProcessor.java
Reads VCAP_SERVICES["ai-models"][0] at startup. Maps credentials.endpoint.openai_api_base and api_key to spring.ai.openai.chat.base-url and chat.api-key. Does nothing locally.
CF VCAP binding
🔧
RedisSslFixPostProcessor.java
config/RedisSslFixPostProcessor.java
Fixes a java-cfenv-boot 3.1.x incompatibility: it sets spring.data.redis.ssl=true (String) but Spring Boot 3.3+ expects a nested Ssl record with spring.data.redis.ssl.enabled. Adds the correct key at highest priority.
Compat fix
